명령어,기능,설명
git init,저장소 초기화,현재 폴더를 Git 저장소로 만듭니다.
git clone <URL>,원격 저장소 복제,원격 저장소의 모든 내용을 로컬로 가져옵니다.
git remote add origin <URL>,원격 저장소 연결,로컬 저장소와 원격 저장소를 연결하고 별명(origin)을 지정합니다.


명령어,기능,설명
git status,상태 확인,"현재 작업 디렉토리의 변경 상태(수정, 스테이징, 추적되지 않음)를 확인합니다."
git add <파일>,스테이징,변경된 파일을 커밋할 준비 영역(스테이징 영역)에 추가합니다.
git add .,전체 스테이징,모든 변경된 파일을 스테이징 영역에 추가합니다.
"git commit -m ""메시지""",커밋,스테이징된 파일을 로컬 저장소에 버전으로 기록합니다. (-m은 메시지 옵션)
"git commit -am ""메시지""",수정 + 커밋,추적 중인 파일의 수정사항을 add와 commit을 한 번에 처리합니다.
git log,이력 확인,커밋 기록을 최신 순으로 확인합니다.
git log --oneline --graph,간결한 이력 확인,커밋 이력을 그래프 형태로 간결하게 보여줍니다.


명령어,기능,설명
git branch,브랜치 목록,로컬에 존재하는 브랜치 목록을 확인합니다.
git branch <새 브랜치 이름>,브랜치 생성,새로운 브랜치를 만듭니다.
git switch <브랜치 이름>,브랜치 이동,해당 브랜치로 작업 위치를 변경합니다. (최신 명령어)
git checkout <브랜치 이름>,브랜치 이동 (구형),해당 브랜치로 이동합니다.
git switch -c <새 브랜치 이름>,생성 및 이동,브랜치를 만들고 바로 그 브랜치로 이동합니다.
git merge <브랜치 이름>,브랜치 병합,지정된 브랜치의 변경 사항을 현재 브랜치로 병합합니다.


명령어,기능,설명
git push origin <브랜치 이름>,원격으로 전송,로컬 커밋을 원격 저장소(origin)로 업로드합니다.
git push -u origin <브랜치 이름>,추적 설정 후 전송,업로드와 동시에 다음부터 git push만 입력해도 되도록 추적 설정을 합니다.
git pull origin <브랜치 이름>,원격 내용 가져오기,원격 저장소의 최신 내용을 가져와서 로컬 브랜치에 병합합니다.
git fetch,원격 정보 확인,"원격 저장소의 최신 커밋/브랜치 정보를 다운로드하지만, 로컬 작업 공간에 병합하지는 않습니다."
git remote -v,원격 연결 확인,현재 연결된 원격 저장소의 이름과 주소(URL)를 확인합니다.


명령어,기능,설명
git restore <파일>,수정 취소,작업 디렉토리에서 수정한 내용을 되돌립니다. (스테이징된 내용은 제외)
git reset HEAD <파일>,스테이징 취소,스테이징 영역에 올린 파일을 스테이징 전 상태로 되돌립니다.
git reset --hard <커밋 ID>,강제 되돌리기,지정된 커밋 ID 상태로 모든 파일과 커밋 이력을 되돌립니다. (주의 필요 - 기록이 사라짐)
git revert <커밋 ID>,안전하게 되돌리기,특정 커밋의 변경 사항을 취소하는 새로운 커밋을 만듭니다. (이력이 남음)

이것은 develop branch입니다.
---------------------------------------------------------------------------------------------------------------
🚀 다른 해결 방법: -u 옵션을 사용하여 첫 푸시
만약 아직 이 브랜치로 한 번도 푸시를 하지 않았다면, push 명령어에 -u 옵션을 추가하여 푸시와 동시에 추적 정보를 설정할 수 있습니다.

Bash

git push -u origin develop
이 명령어는 로컬 develop 브랜치를 origin/develop로 푸시하고, 앞으로는 이 둘을 연결하여 추적하겠다는 의미를 동시에 가집니다.

---------------------------------------------------------------------------------------------------------------

🛠️ 해결 방법: 추적 정보 설정
이 문제를 해결하려면 현재 로컬 브랜치(develop)가 원격 저장소(origin)의 같은 이름 브랜치(develop)를 추적하도록 명시적으로 설정해 주어야 합니다.

터미널이나 VS Code의 통합 터미널에서 다음 명령어를 한 번만 실행해 주세요.

Bash

git branch --set-upstream-to=origin/develop develop
명령어 해설:
git branch: 브랜치 관련 명령어입니다.

--set-upstream-to=origin/develop: 원격 저장소 origin에 있는 develop 브랜치를 업스트림(추적 대상)으로 설정하겠다는 의미입니다.

develop: 현재 로컬 브랜치(develop)를 지정합니다.
